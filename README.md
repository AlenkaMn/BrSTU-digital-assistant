# BrSTU-digital-assistant

### Описание проекта.
**BrSTU Digital Assistant** - интеллектуальный цифровой ассистент для поддержки студентов, сотрудников и абитуриентов БрГТУ, разработанный на основе технологии RAG и методов искусственного интеллекта. Система предназначена для автоматизации поиска, анализа и выдачи справочной и учебной информации, а также для снижения административной нагрузки на сотрудников университета. Ассистент использует базу знаний БрГТУ, гибридный поиск по данным и языковую модель для формирования точных и контекстно обоснованных ответов. Интеграция с Telegram-ботом обеспечивает круглосуточный доступ к информации без необходимости установки отдельного приложения.


P.S. т.к. на данный момент я ещё не защищала дипломный проект, поэтому в общий досуп могу выложить только базовую версию своей системы.
А также под капотом Embedder от Яндекс и LLM - YandexGPT - разработки Яндекса также отлично подходят для моей задачи.


### Проект имеет следующую структуру:

```
BrSTU-digital-assistant/
├── data/
├── images/
├── logs/
├── src/
│   ├── adapters/
│   │   └── tg/
│   │       └── bot.py
│   ├── config/
│   │   └── config.yaml
│   ├── core/
│   │   ├── __init__.py
│   │   └── LLMClient.py
│   ├── infrastructure/
│   │   ├── llm/
│   │   │   └── YandexLLMClient.py
│   │   ├── rag/
│   │   │   ├── __init__.py
│   │   │   ├── Embedder.py
│   │   │   ├── Retriever.py
│   │   │   ├── Rewriter.py
│   │   │   └── Store.py
│   │   └── data_base.py
```

`data/` - папка для хранения данных базы знаний: текстовых чанков, JSON-файлов и заранее рассчитанных эмбеддингов.

`images/` - папка для хранения изображений, связанных с мультимодальной базой знаний.

`logs/` - папка для хранения логов работы системы: запуск бота, обработка запросов, ошибки и действия с БД.

`src/` - основная папка с исходным кодом проекта.

`adapters/` - папка с адаптерами для взаимодействия системы с внешними интерфейсами.

`tg/` - папка с реализацией Telegram-интерфейса ассистента.

`bot.py` - файл Telegram-бота, содержащий обработчики сообщений, команд пользователя и администратора.

`config/` - папка с конфигурационными файлами проекта.

`config.yaml` - файл настроек проекта: токены и ключи.

`core/` - папка с базовыми интерфейсами и абстракциями проекта.

`LLMClient.py` - базовый интерфейс для работы с языковой моделью.

`infrastructure/` - папка с технической реализацией основных компонентов системы.

`llm/` - папка с реализацией клиента языковой модели.

`YandexLLMClient.py` - клиент для обращения к YandexGPT и получения ответов от языковой модели.

`rag/` - папка с компонентами RAG-модуля.


`Embedder.py` - модуль получения эмбеддингов текстов и пользовательских запросов через Yandex Embeddings.

`Retriever.py` - модуль поиска релевантных фрагментов базы знаний с использованием гибридного поиска: эмбеддинги, BM25 и метод Борда.

`Rewriter.py` - модуль предварительной обработки запроса: приведение к нижнему регистру, удаление лишних пробелов и последовательностей цифр без удаления стоп-слов.

`Store.py` - модуль загрузки и подготовки документов базы знаний из файлов.

`data_base.py` - модуль работы с базой данных: таблицы `users` и `messages`, сохранение истории диалогов, получение последних сообщений и учёт токсичных запросов.



Для запуска системы необходимо:
1. создать виртуальное окружение и скачать необходимые пакеты, описанные в reuirements.txt.
2. запустить файл main.py
